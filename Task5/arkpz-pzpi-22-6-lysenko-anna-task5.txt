3. Хід роботи:
3.1 Розгорнути реалізовану програмну систему.
Для розгортання проєкту було використано хостинг-платформу Somee та її Free hosting package.
Після реєстрації на платформі було створено вебсайт з визначеним доменом (див. рис. 3.1). 

Рисунок 3.1 – Характеристики створеного вебсайту
     
Додано та налаштовано базу даних для взаємодії (див. рис. 3.2).

Рисунок 3.2 – Характеристики створеної БД

Перед створенням публікаційної версії проєкту було змінено посилання на базу даних у файлі appsettings.json.
    1	"ConnectionStrings": {
    2	  "DefaultConnection": "workstation id=GasLeakDB.mssql.somee.com;packet size=4096;user id=annly_0_0_SQLLogin_1;pwd=3xj6irjj8s;data source=GasLeakDB.mssql.somee.com;persist security info=False;initial catalog=GasLeakDB;TrustServerCertificate=True"
    3     }
Тепер замість локальної бази даних використовується БД, створена на хостинговій платформі. Для переносу локальної бази даних використовувалися згенеровані скрипти.
Після виконання необхідних налаштувань для розгортання системи створено публікацію проєкту (див. рис. 3.3), яка містить всі необхідні файли: виконувані файли (.dll),  налаштування (appsettings.json), файл web.config (потрібен для хостингу) та інші статичні файли.

Рисунок 3.3 – Публікація проєкту
     
Всі отримані файли проєкту після публікування були перенесені до File Manager на хостинговій платформі (див. рис. 3.4). Після цього можна перевіряти роботу системи, перейшовши на домен: http://GasDecSystem1.somee.com 

Рисунок 3.4 – Файли проєкту
     
При розробці проєкту використовувався Entity Framework Core, тому структура міграції даних була створена після виконання команди dotnet ef migrations add InitialMigration (можна переглянути у папці Migrations). Оновлення відбувається завдяки виконанню команди dotnet ef database update.
